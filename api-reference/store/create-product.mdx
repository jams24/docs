## Overview

Adds a new product to your store. Products are priced in **USDT** and automatically converted to the customer's chosen cryptocurrency at checkout. A URL slug is generated from the product name.

For digital products, you can attach **digital content** (file URLs, license keys, instructions) that will be automatically delivered to the customer when payment is confirmed.

## Authentication

<ParamField header="X-API-Key" type="string" required>
  Your InventPay API key
</ParamField>

## Request Body

<ParamField body="name" type="string" required>
  Product name (1-200 characters) **Example:** `"Premium API Access"`
</ParamField>

<ParamField body="description" type="string" optional>
  Product description (max 5000 characters) **Example:** `"Unlimited API calls for 30 days"`
</ParamField>

<ParamField body="price" type="number" required>
  Price in USDT (0.01 - 1,000,000) **Example:** `49.99`
</ParamField>

<ParamField body="images" type="string[]" optional>
  Array of HTTPS image URLs (max 10) **Example:** `["https://example.com/product.jpg"]`
</ParamField>

<ParamField body="stock" type="integer" optional>
  Available quantity. Omit for unlimited stock. **Example:** `100`
</ParamField>

<ParamField body="variants" type="array" optional>
  Variant options **Example:** `[{"name": "Plan", "options": ["Basic", "Pro"]}]`
</ParamField>

<ParamField body="metadata" type="object" optional>
  Custom key-value data attached to the product
</ParamField>

<ParamField body="contentType" type="string" optional>
  Product content type. `GENERAL` (default) for standard digital products, `SKILL` for AI agent skill files. **Example:** `"SKILL"`
</ParamField>

<ParamField body="skillContent" type="string" optional>
  Raw SKILL.md file content (markdown with YAML frontmatter). Only used when `contentType` is `SKILL`. The backend automatically parses the YAML frontmatter and stores it as `skillMeta` for storefront display. The raw content is delivered to buyers after payment. Max 100,000 characters.
  **Example:** `"---\nname: my-skill\nauthor: dev\nversion: 1.0.0\ntags:\n  - automation\n---\n\n# Skill Instructions\n..."`
</ParamField>

<ParamField body="digitalContent" type="object" optional>
  Digital delivery content — automatically delivered to customer after payment. Can be used alongside `skillContent` for skill products (e.g., add a download URL or setup instructions).
  <Expandable title="digitalContent properties">
    <ParamField body="digitalContent.fileUrl" type="string" optional>
      Download URL for the digital file **Example:** `"https://cdn.example.com/ebook.pdf"`
    </ParamField>
    <ParamField body="digitalContent.fileName" type="string" optional>
      Display name for the file **Example:** `"Premium-Guide-v2.pdf"`
    </ParamField>
    <ParamField body="digitalContent.fileSize" type="number" optional>
      File size in bytes **Example:** `2048000`
    </ParamField>
    <ParamField body="digitalContent.licenseKey" type="string" optional>
      License or activation key **Example:** `"XXXX-XXXX-XXXX-XXXX"`
    </ParamField>
    <ParamField body="digitalContent.instructions" type="string" optional>
      Setup or access instructions **Example:** `"Download the file and run setup.exe"`
    </ParamField>
    <ParamField body="digitalContent.textContent" type="string" optional>
      Inline text content delivered directly **Example:** `"Your access code is: ABC123"`
    </ParamField>
  </Expandable>
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the request was successful
</ResponseField>

<ResponseField name="data" type="object">
  Product details
  <Expandable title="data properties">
    <ResponseField name="id" type="string">
      Unique product identifier
    </ResponseField>
    <ResponseField name="name" type="string">
      Product name
    </ResponseField>
    <ResponseField name="slug" type="string">
      URL-safe product slug
    </ResponseField>
    <ResponseField name="price" type="string">
      Price in USDT (decimal string)
    </ResponseField>
    <ResponseField name="stock" type="integer | null">
      Available quantity or `null` for unlimited
    </ResponseField>
    <ResponseField name="productType" type="string">
      Product type: `DIGITAL`
    </ResponseField>
    <ResponseField name="contentType" type="string">
      Content type: `GENERAL` or `SKILL`
    </ResponseField>
    <ResponseField name="skillMeta" type="object | null">
      Parsed YAML frontmatter from `skillContent` (only for SKILL products). Contains `name`, `description`, `author`, `version`, `tags`, etc.
    </ResponseField>
    <ResponseField name="isActive" type="boolean">
      Whether the product is visible
    </ResponseField>
    <ResponseField name="createdAt" type="string">
      Creation timestamp (ISO 8601)
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl -X POST https://api.inventpay.io/v1/store/manage/products \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Claude Skills Guide",
    "description": "Complete guide to building AI agent skills",
    "price": 19.99,
    "images": ["https://example.com/guide-cover.jpg"],
    "digitalContent": {
      "fileUrl": "https://cdn.example.com/claude-skills-guide.pdf",
      "fileName": "Claude-Skills-Guide-v2.pdf",
      "fileSize": 2048000,
      "instructions": "Download the PDF and open with any PDF reader"
    }
  }'
```

```javascript JavaScript/TypeScript
const response = await fetch("https://api.inventpay.io/v1/store/manage/products", {
  method: "POST",
  headers: {
    "X-API-Key": "YOUR_API_KEY",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    name: "Claude Skills Guide",
    description: "Complete guide to building AI agent skills",
    price: 19.99,
    images: ["https://example.com/guide-cover.jpg"],
    digitalContent: {
      fileUrl: "https://cdn.example.com/claude-skills-guide.pdf",
      fileName: "Claude-Skills-Guide-v2.pdf",
      fileSize: 2048000,
      instructions: "Download the PDF and open with any PDF reader",
    },
  }),
});

const product = await response.json();
console.log(product.data.id); // Use this ID for checkout
```

```python Python
import requests

response = requests.post(
    "https://api.inventpay.io/v1/store/manage/products",
    headers={"X-API-Key": "YOUR_API_KEY"},
    json={
        "name": "Claude Skills Guide",
        "description": "Complete guide to building AI agent skills",
        "price": 19.99,
        "images": ["https://example.com/guide-cover.jpg"],
        "digitalContent": {
            "fileUrl": "https://cdn.example.com/claude-skills-guide.pdf",
            "fileName": "Claude-Skills-Guide-v2.pdf",
            "fileSize": 2048000,
            "instructions": "Download the PDF and open with any PDF reader"
        }
    }
)

product = response.json()
print(product["data"]["id"])
```

### Skill Product Example

To sell an AI agent skill file, set `contentType` to `SKILL` and provide the raw SKILL.md content in `skillContent`:

<CodeGroup>

```bash cURL
curl -X POST https://api.inventpay.io/v1/store/manage/products \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Code Review Pro Skill",
    "description": "AI skill that catches security vulnerabilities",
    "price": 8.99,
    "contentType": "SKILL",
    "skillContent": "---\nname: code-review-pro\nauthor: devtools\nversion: 2.0.0\ntags:\n  - security\n  - code-review\n---\n\n# Code Review Pro\n\nInstructions for the agent..."
  }'
```

```javascript JavaScript/TypeScript
const response = await fetch("https://api.inventpay.io/v1/store/manage/products", {
  method: "POST",
  headers: {
    "X-API-Key": "YOUR_API_KEY",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    name: "Code Review Pro Skill",
    description: "AI skill that catches security vulnerabilities",
    price: 8.99,
    contentType: "SKILL",
    skillContent: `---
name: code-review-pro
author: devtools
version: 2.0.0
tags:
  - security
  - code-review
---

# Code Review Pro

Instructions for the agent...`,
  }),
});
```

```python Python
import requests

response = requests.post(
    "https://api.inventpay.io/v1/store/manage/products",
    headers={"X-API-Key": "YOUR_API_KEY"},
    json={
        "name": "Code Review Pro Skill",
        "description": "AI skill that catches security vulnerabilities",
        "price": 8.99,
        "contentType": "SKILL",
        "skillContent": "---\nname: code-review-pro\nauthor: devtools\nversion: 2.0.0\ntags:\n  - security\n  - code-review\n---\n\n# Code Review Pro\n\nInstructions for the agent..."
    }
)
```

</CodeGroup>

The backend automatically parses the YAML frontmatter and the storefront displays the skill metadata (name, version, author, tags) as a rich product card with a purple "SKILL" badge. The raw `skillContent` is never exposed publicly — it is delivered to the buyer after payment.

## Error Codes

| Status Code | Description |
| ----------- | ----------- |
| 201 | Product created successfully |
| 400 | Invalid request parameters |
| 401 | Invalid or missing API key |
| 404 | No store found (create a store first) |
| 429 | Rate limit exceeded |
