## Overview

Creates an order and generates a crypto payment address. This is a **public endpoint** — no API key required. The customer sends crypto to the generated address, and the order automatically transitions to `PAID` when payment is confirmed on-chain.

For digital products, the order then automatically transitions to `COMPLETED` with a download token generated for the customer.

**What happens during checkout:**

1. Products and stock are validated
2. USDT total is converted to the selected cryptocurrency at real-time rates
3. A unique HD wallet address is generated
4. Order + Payment are created atomically (stock is decremented)
5. Returns the payment address, amount, and order details

**After payment is confirmed:**

6. Order status transitions to `PAID`
7. A unique download token is generated
8. Order status transitions to `COMPLETED`
9. Delivery email is sent (if `customerEmail` was provided)
10. Customer accesses downloads via `/download/{token}`

## Path Parameters

<ParamField path="slug" type="string" required>
  Store slug (from the store URL)
</ParamField>

## Request Body

<ParamField body="items" type="array" required>
  Array of items to purchase (1-50 items)
  <Expandable title="item properties">
    <ParamField body="items[].productId" type="string" required>
      Product ID
    </ParamField>
    <ParamField body="items[].quantity" type="integer" required>
      Quantity (1-100)
    </ParamField>
    <ParamField body="items[].variant" type="object" optional>
      Selected variant options
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="currency" type="string" optional>
  Cryptocurrency for payment: `BTC`, `ETH`, `LTC`, `USDT_ERC20`, `USDT_BEP20` (default: `USDT_ERC20`)
</ParamField>

<ParamField body="customerEmail" type="string" optional>
  Customer's email for delivery notification. **Recommended** — the delivery email with download link is sent to this address.
</ParamField>

<ParamField body="customerName" type="string" optional>
  Customer's name (max 100 characters)
</ParamField>

## Example Request

<CodeGroup>

```bash cURL
curl -X POST https://api.inventpay.io/v1/store/my-crypto-shop/checkout \
  -H "Content-Type: application/json" \
  -d '{
    "items": [
      { "productId": "cm1prod123abc", "quantity": 1 }
    ],
    "currency": "ETH",
    "customerEmail": "customer@example.com"
  }'
```

```javascript JavaScript/TypeScript
const response = await fetch(
  "https://api.inventpay.io/v1/store/my-crypto-shop/checkout",
  {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      items: [{ productId: "cm1prod123abc", quantity: 1 }],
      currency: "ETH",
      customerEmail: "customer@example.com",
    }),
  }
);

const { data } = await response.json();
console.log(`Send ${data.payment.amount} ${data.payment.currency} to ${data.payment.address}`);
// Poll order status to detect when payment confirms
console.log(`Order number: ${data.order.orderNumber}`);
```

```python Python
import requests

response = requests.post(
    "https://api.inventpay.io/v1/store/my-crypto-shop/checkout",
    json={
        "items": [{"productId": "cm1prod123abc", "quantity": 1}],
        "currency": "ETH",
        "customerEmail": "customer@example.com"
    }
)

data = response.json()["data"]
print(f"Send {data['payment']['amount']} {data['payment']['currency']} to {data['payment']['address']}")
```

## After Payment

Once the customer sends crypto and the payment is confirmed on-chain:

1. **Order status** transitions: `PENDING` → `PAID` → `COMPLETED`
2. **Download token** is generated and attached to the order
3. **Delivery email** is sent to the customer (if email was provided)
4. Customer can access downloads at `https://inventpay.io/download/{token}`

Use the [Order Status](/api-reference/store/order-status) endpoint to poll for payment confirmation, or the [Download](/api-reference/store/download) endpoint to retrieve digital content.

## Error Codes

| Status Code | Description |
| ----------- | ----------- |
| 201 | Order created, payment address generated |
| 400 | Invalid request (missing items, invalid product ID) |
| 404 | Store not found |
| 409 | Insufficient stock |
| 503 | Store temporarily unavailable |
