---

## Overview

Updates an order's fulfillment status. Only valid transitions are allowed (e.g., `PAID → PROCESSING → SHIPPED → COMPLETED`).

**Valid transitions:**

| From | Allowed To |
| ---- | ---------- |
| `PAID` | `PROCESSING`, `CANCELLED` |
| `PROCESSING` | `SHIPPED`, `CANCELLED` |
| `SHIPPED` | `COMPLETED` |
| `COMPLETED` | `REFUNDED` |

<Info>
The `PENDING → PAID` transition happens automatically when crypto payment is confirmed on-chain.
</Info>

## Authentication

<ParamField header="X-API-Key" type="string" required>
  Your InventPay API key
</ParamField>

## Path Parameters

<ParamField path="id" type="string" required>
  Order ID
</ParamField>

## Request Body

<ParamField body="status" type="string" required>
  New order status: `PROCESSING`, `SHIPPED`, `COMPLETED`, `CANCELLED`, `REFUNDED`
</ParamField>

<ParamField body="notes" type="string" optional>
  Optional note (max 500 characters). Use for tracking numbers, cancellation reasons, etc.
</ParamField>

## Error Codes

| Status Code | Description |
| ----------- | ----------- |
| 200 | Status updated successfully |
| 400 | Invalid status transition |
| 401 | Invalid or missing API key |
| 404 | Order or store not found |
| 429 | Rate limit exceeded |
