## Overview

The InventPay MCP Server lets AI agents (Claude, Cursor, Windsurf, and any MCP-compatible client) manage crypto payments, stores, products, and orders through natural language.

With the MCP server, an AI agent can:
- Create and manage a crypto storefront
- Add digital products with automatic delivery
- Accept crypto payments and track balances
- Monitor orders and revenue
- Process withdrawals

## Installation

```bash
npm install -g inventpay-mcp
```

## Setup

### 1. Get your API Keys

1. Sign up at [inventpay.io](https://inventpay.io)
2. Go to **Dashboard → Settings**
3. Copy your **API key** (`pk_live_...`) — this is your main key for all operations
4. *(Optional)* Generate a **Withdrawal API key** (`wk_live_...`) under **Withdrawal API Key** — only needed if you want the AI agent to withdraw funds to external wallets

### 2. Configure your AI tool

<Tabs>

<Tab title="Claude Desktop">

## Available Tools

### Payments

| Tool | Description |
|------|-------------|
| `create_payment` | Create a crypto payment with a unique wallet address, QR code, and invoice URL |
| `create_invoice` | Create a multi-currency invoice — customer chooses which crypto to pay with |
| `get_payment_status` | Check payment status, confirmations, and transaction details |

### Balances

| Tool | Description |
|------|-------------|
| `get_balances` | Get all cryptocurrency balances (available, pending, total earned, total withdrawn) |

### Withdrawals

| Tool | Description |
|------|-------------|
| `create_withdrawal` | Withdraw funds to an external wallet address (requires `INVENTPAY_WITHDRAWAL_KEY`) |
| `get_withdrawal` | Check withdrawal status and transaction hash |

### Store Management

| Tool | Description |
|------|-------------|
| `create_store` | Create a crypto-powered storefront with a unique URL |
| `get_store` | Get store details, product count, and order count |
| `update_store` | Update store name, description, logo, banner, or toggle visibility |

### Products

| Tool | Description |
|------|-------------|
| `create_product` | Add a digital product with price, images, and automatic delivery content |
| `list_products` | List products with search and pagination |
| `update_product` | Update product details, price, stock, or digital content |
| `delete_product` | Deactivate a product from your store |

### Key Pool

| Tool | Description |
|------|-------------|
| `add_product_keys` | Upload unique keys/codes to a product's pool (license keys, gift cards, etc.) |
| `list_product_keys` | List keys in the pool with status filter (AVAILABLE, ASSIGNED, REVOKED) |
| `get_key_pool_stats` | Get pool statistics: available, assigned, revoked, and total counts |

### Orders

| Tool | Description |
|------|-------------|
| `list_orders` | List orders with optional status filter and pagination |
| `get_order` | Get full order details including items, payment, and customer info |
| `update_order_status` | Update fulfillment status (PROCESSING, SHIPPED, COMPLETED, etc.) |

### Analytics

| Tool | Description |
|------|-------------|
| `get_store_analytics` | Get order counts, revenue breakdown by status, and product stats |

## Digital Product Delivery

When you create a product with `digitalContent`, InventPay automatically delivers it to the customer after payment:

```
Create a product called "Claude Skills Guide" for $19.99 with:
- Download URL: https://cdn.example.com/guide.pdf
- File name: Claude-Skills-Guide.pdf
- Instructions: "Open with any PDF reader"
```

The AI agent will call `create_product` with the appropriate `digitalContent` fields. After a customer pays, they automatically receive a download link.

## Environment Variables

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `INVENTPAY_API_KEY` | Yes | — | Your merchant API key (`pk_live_...`). Used for **all API operations**: creating payments/invoices, managing your store, products, orders, checking balances, and reading data. This is your main key for everything except withdrawals. |
| `INVENTPAY_WITHDRAWAL_KEY` | No | — | Withdrawal-only API key (`wk_live_...`). Used **exclusively** for withdrawing funds to external wallet addresses. This key cannot be used for any other operation. Generate from **Dashboard → Settings → Withdrawal API Key**. Without this, `create_withdrawal` will be disabled. |
| `INVENTPAY_BASE_URL` | No | `https://api.inventpay.io` | API base URL |

<Warning>
  **Do not confuse the two keys.** The API key (`INVENTPAY_API_KEY`) handles all endpoint operations — payments, invoices, store management, balances, etc. The withdrawal key (`INVENTPAY_WITHDRAWAL_KEY`) is a separate, restricted key that **only** authorizes sending funds out to external wallets. Both keys are required together when making a withdrawal request.
</Warning>

## Agent Instructions

Copy the prompt below into your agent's system instructions (CLAUDE.md, custom instructions, or system prompt) so it knows how to use the InventPay tools effectively.

<Tip>
  For Claude Code, save this as `CLAUDE.md` in your project root. For Claude Desktop or Cursor, paste it into your custom instructions or system prompt.
</Tip>

There are two ways agents can interact with InventPay:

1. **MCP Server** (recommended) — install the MCP server and tools are available automatically
2. **Direct API** — call REST endpoints with your API key in the `X-API-Key` header

<Tab title="For MCP Agents">

```text MCP Agent Prompt
# InventPay MCP Tools

You have access to InventPay MCP tools for managing crypto payments, stores, and withdrawals.

## API Keys
- INVENTPAY_API_KEY (pk_live_...) — Your main merchant key. Used for ALL operations: creating payments, invoices, managing store/products/orders, checking balances, etc.
- INVENTPAY_WITHDRAWAL_KEY (wk_live_...) — Withdrawal-only key. Used EXCLUSIVELY for withdrawing funds to external wallet addresses. Cannot be used for anything else. Both keys are required together for withdrawal requests.

## Available Tools

Payments:
- create_payment — Create a crypto payment. Params: amount (number), currency (BTC|ETH|LTC|USDT_ERC20|USDT_BEP20|SOL|USDC_SOL|USDC_BEP20), amountCurrency (USD|USDT|SOL), orderId, description, callbackUrl, expirationMinutes
- create_invoice — Create a multi-currency invoice (customer picks crypto). Params: amount, amountCurrency, orderId, description, expirationMinutes
- get_payment_status — Check payment status. Params: paymentId

Balances:
- get_balances — Get all crypto balances (available, pending, locked, earned, withdrawn)

Withdrawals:
- create_withdrawal — Withdraw to external wallet (requires INVENTPAY_WITHDRAWAL_KEY). Params: amount, currency, destinationAddress, description
- get_withdrawal — Check withdrawal status. Params: withdrawalId

Store:
- create_store — Create a storefront. Params: name, description
- get_store — Get store details
- update_store — Update store. Params: name, description, logo, banner, isPublished

Products:
- create_product — Add product. Params: name, price, currency, description, stock, images, digitalContent (fileUrl, fileName, instructions, licenseKey, textContent)
- list_products — List products. Params: page, limit, search
- update_product — Update product. Params: productId, then same as create
- delete_product — Deactivate product. Params: productId

Key Pool:
- add_product_keys — Upload unique keys/codes to a product pool. Params: productId, keys (string[]), label
- list_product_keys — List keys in pool. Params: productId, page, limit, status (AVAILABLE|ASSIGNED|REVOKED)
- get_key_pool_stats — Get pool stats (available, assigned, revoked, total). Params: productId

Orders:
- list_orders — List orders. Params: page, limit, status
- get_order — Get order details. Params: orderId
- update_order_status — Update fulfillment. Params: orderId, status (PROCESSING|SHIPPED|COMPLETED|CANCELLED)

Analytics:
- get_store_analytics — Revenue breakdown, order counts, product stats

## Important URLs
- Store page: https://inventpay.io/store/{slug}
- Product page: https://inventpay.io/store/{slug}/product/{productSlug} (NOT /store/{slug}/{productSlug} — the /product/ segment is required)
- Invoice/payment page: https://inventpay.io/invoice/{paymentId}
- Do NOT omit /product/ from product URLs — /store/{slug}/{productSlug} will NOT work

## Important: Correct Endpoint Paths
- Payment creation: POST /v1/create_payment (NOT /v1/payments, NOT /v1/invoices)
- Invoice creation: POST /v1/create_invoice (NOT /v1/invoices, NOT /v1/store/manage/invoices)
- Store checkout: POST /v1/store/{slug}/checkout (uses the store's URL slug, NOT a store ID)
- Do NOT guess endpoint paths — use only the exact paths listed above

## Guidelines
- All amounts in USD unless specified otherwise
- Supported cryptos: BTC, ETH, LTC, USDT_ERC20, USDT_BEP20, SOL, USDC_SOL, USDC_BEP20
- Payment expiration: 5-1440 minutes (default 30)
- Always confirm withdrawal details with the user before executing
- Withdrawals require BOTH keys (API key + withdrawal key) and do NOT require 2FA — just call the endpoint directly
- The withdrawal key (wk_live_...) can ONLY be used for withdrawals — never use it for other operations
- The API key (pk_live_...) is used for everything else — payments, invoices, store, products, orders, balances
- Use create_payment for single-crypto, create_invoice for multi-crypto choice
- Digital products auto-deliver after payment via digitalContent field
```

# InventPay API

You can manage crypto payments, stores, and withdrawals via the InventPay REST API.

## Base URL
https://api.inventpay.io

## Authentication & API Keys
There are two separate keys with distinct purposes:

1. API Key (pk_live_...) — Your main merchant key for ALL endpoint operations (payments, invoices, store, products, orders, balances). Include in every request:
  Header: X-API-Key: YOUR_API_KEY
  Content-Type: application/json

2. Withdrawal Key (wk_live_...) — Restricted key used EXCLUSIVELY for withdrawing funds to external wallets. Cannot be used for any other operation. For withdrawal requests, include BOTH keys:
  Header: X-API-Key: YOUR_API_KEY
  Header: X-Withdrawal-Key: YOUR_WITHDRAWAL_KEY

## IMPORTANT: Correct Endpoint Paths
- Payment creation is POST /v1/create_payment — NOT /v1/payments
- Invoice creation is POST /v1/create_invoice — NOT /v1/invoices or /v1/store/manage/invoices
- Store checkout is POST /v1/store/{slug}/checkout — uses the store URL slug (e.g., "my-store"), NOT a store ID
- Do NOT guess or invent endpoint paths. Use ONLY the exact paths listed below.

## API Endpoints

Payments:
- POST /v1/create_payment — Create a crypto payment. Body: { amount, currency (BTC|ETH|LTC|USDT_ERC20|USDT_BEP20|SOL|USDC_SOL|USDC_BEP20), amountCurrency (USD|USDT|SOL), orderId, description, callbackUrl, expirationMinutes }
- POST /v1/create_invoice — Create a multi-currency invoice. Body: { amount, amountCurrency, orderId, description, expirationMinutes }
- GET /v1/invoice/{paymentId}/status — Check payment status (no auth required)

Store Checkout (public, no auth):
- POST /v1/store/{slug}/checkout — Create a store order. Body: { items: [{ productId, quantity, variant }], customerEmail, customerName, currency }

Balances:
- GET /v1/merchant/balance — Get all crypto balances
- GET /v1/merchant/balance/{currency} — Get balance for specific currency

Withdrawals:
- POST /v1/merchant/withdrawal/create — Withdraw to external wallet (requires X-Withdrawal-Key header). Body: { amount, currency, destinationAddress, description }
- GET /v1/withdrawal/{withdrawalId} — Check withdrawal status

Store Management (auth required):
- POST /v1/store/manage — Create a store. Body: { name, description }
- GET /v1/store/manage — Get store details
- PUT /v1/store/manage — Update store. Body: { name, description, logo, banner, isPublished }

Products:
- POST /v1/store/manage/products — Add product. Body: { name, price, currency, description, stock, images, digitalContent: { fileUrl, fileName, instructions, licenseKey, textContent } }
- GET /v1/store/manage/products — List products. Query: ?page=1&limit=20&search=term
- PUT /v1/store/manage/products/{productId} — Update product
- DELETE /v1/store/manage/products/{productId} — Delete product

Key Pool (unique keys per customer):
- POST /v1/store/manage/products/{productId}/keys — Upload keys. Body: { keys: ["KEY1", "KEY2", ...], label: "Batch #1" }
- GET /v1/store/manage/products/{productId}/keys — List keys. Query: ?page=1&limit=50&status=AVAILABLE
- GET /v1/store/manage/products/{productId}/keys/stats — Get pool stats (available, assigned, revoked, total)
- DELETE /v1/store/manage/products/{productId}/keys — Remove all available (unassigned) keys

Orders:
- GET /v1/store/manage/orders — List orders. Query: ?page=1&limit=20&status=PAID
- GET /v1/store/manage/orders/{orderId} — Get order details
- PUT /v1/store/manage/orders/{orderId}/status — Update order status. Body: { status } (PROCESSING|SHIPPED|COMPLETED|CANCELLED)

## Important URLs
- Store page: https://inventpay.io/store/{slug}
- Product page: https://inventpay.io/store/{slug}/product/{productSlug} (NOT /store/{slug}/{productSlug} — the /product/ segment is required)
- Invoice/payment page: https://inventpay.io/invoice/{paymentId}
- Do NOT omit /product/ from product URLs — /store/{slug}/{productSlug} will NOT work

## Guidelines
- All amounts in USD unless specified otherwise
- Supported cryptos: BTC, ETH, LTC, USDT_ERC20, USDT_BEP20, SOL, USDC_SOL, USDC_BEP20
- Payment expiration: 5-1440 minutes (default 30)
- Always confirm withdrawal details with the user before executing
- Withdrawals require BOTH headers (X-API-Key + X-Withdrawal-Key) and do NOT require 2FA — just call the endpoint directly
- The withdrawal key (wk_live_...) can ONLY be used for withdrawals — never use it for other operations
- The API key (pk_live_...) is used for everything else — payments, invoices, store, products, orders, balances
- Use /v1/create_payment for single-crypto, /v1/create_invoice for multi-crypto choice
- Store checkout uses the store slug in the URL path, NOT a store ID
- Digital products auto-deliver after payment via digitalContent field
- All responses follow: { success: true/false, data: {...}, message: "..." }
